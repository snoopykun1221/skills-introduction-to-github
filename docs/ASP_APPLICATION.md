## 各ASPごとの登録ポイント

### 1. A8.net (https://www.a8.net/)
- **審査**: なし(誰でも登録可)
- **必要情報**: 氏名・住所・電話・銀行口座・サイト情報
- **登録後すぐ提携できる主要案件**:
  - アソビュー！(体験予約)
  - HOT PEPPER グルメ
  - エアトリ(航空券)
  - 楽天トラベル
- **取得するもの**:
  - メディアID(=`a8_id_*` に該当)
  - 各案件の `a8mat=` パラメータ

### 2. もしもアフィリエイト (https://af.moshimo.com/)
- **審査**: あり(数日)
- **必要情報**: サイト情報・銀行口座
- **強み**: 楽天市場・Amazon を W登録 で扱える、W報酬制度
- **取得するもの**: もしも ID(`moshimo_id`)

### 3. バリューコマース (https://www.valuecommerce.ne.jp/)
- **審査**: あり(やや厳しい・記事数10本程度欲しい)
- **必要情報**: サイト情報
- **強み**: 楽天トラベル・じゃらん・ぐるなび・食べログ・一休 すべて扱える
- **取得するもの**:
  - sid(管理画面の「サイト一覧」)→ `valuecommerce_sid`
  - 各案件の pid(「広告主検索」→提携→広告原稿) → `vc_pid_*`

### 4. アクセストレード (https://www.accesstrade.ne.jp/)
- **審査**: あり
- **強み**: Booking.com・agoda・Trip.com など海外OTA
- **インバウンド対応する場合は必須**

### 5. Amazon アソシエイト (https://affiliate.amazon.co.jp/)
- **審査**: 仮承認 → 180日以内に3件成約で本承認
- **必要情報**: サイト情報・銀行口座
- **取得するもの**: トラッキングID(末尾`-22`) → `amazon_tag`

### 6. 楽天アフィリエイト (https://affiliate.rakuten.co.jp/)
- **審査**: なし(楽天会員ならすぐ)
- **報酬**: 楽天キャッシュ or 現金
- **もしも経由で登録すると現金化しやすい**

### 7. Google AdSense (https://www.google.com/adsense/)
- **審査**: 厳しい(記事数 20 本以上、独自性、ポリシー遵守)
- **準備**: プライバシーポリシー(✅作成済み `policy.html`)、お問い合わせ(✅`contact.html`)、運営者情報(✅`about.html`)
- **取得するもの**: パブリッシャー ID `ca-pub-XXXXXXXXXXXXXXXX`

## 登録の推奨順序

1. **A8.net** ← まず登録(審査なし、即提携可能)
2. **もしもアフィリエイト** ← 楽天/Amazon の窓口に
3. **Amazon アソシエイト** ← 仮承認だけ取得(本承認は3件成約後)
4. **楽天アフィリエイト** ← もしも経由なら省略可
5. **バリューコマース** ← サイトに10記事ほど追加してから審査
6. **Google AdSense** ← サイト充実後に申請(最低20記事程度)
7. **アクセストレード** ← インバウンド狙うなら

## ID取得後の作業

`docs/js/data.js` の冒頭 `AFF_IDS` を編集:

```js
const AFF_IDS = {
  valuecommerce_sid:    "実際のsid",
  vc_pid_jalan:         "実際のpid",
  vc_pid_rakuten:       "実際のpid",
  vc_pid_gnavi:         "実際のpid",
  vc_pid_tabelog:       "実際のpid",
  vc_pid_ikyu:          "実際のpid",
  a8_id_asoview:        "実際のa8mat",
  moshimo_id:           "実際のもしもID",
  amazon_tag:           "実際のtag-22"
};
```

`docs/index.html` の冒頭で:
- `GA4_MEASUREMENT_ID` → 実際の `G-XXXXXXXXXX`
- `ADSENSE_CLIENT` → 実際の `ca-pub-XXXXXXXXXXXXXXXX`

を置換すれば公開準備完了です。
